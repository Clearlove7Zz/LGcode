Iako Loongcode moze raditi direktno na Windowsu, preporucujemo [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) za najbolje iskustvo. WSL daje Linux okruzenje koje glatko radi sa Loongcode funkcijama.

:::tip[Za┼Īto WSL?]
WSL nudi bolje performanse fajl sistema, punu terminalsku podrsku i kompatibilnost s razvojnim alatima na koje se Loongcode oslanja.
:::

---

## Postavljanje

<Steps>

1. **Instalirajte WSL**

   Ako vec niste, [instalirajte WSL](https://learn.microsoft.com/en-us/windows/wsl/install) prema zvanicnom Microsoft vodicu.

2. **Instalirajte Loongcode u WSL-u**

   Kad je WSL spreman, otvorite WSL terminal i instalirajte Loongcode jednom od [metoda instalacije](/docs/).

   ```bash
   curl -fsSL https://modelhub.lgdg.cc/install | bash
   ```

3. **Koristite Loongcode iz WSL-a**

   Idite u direktorij projekta (Windows fajlovima pristupate preko `/mnt/c/`, `/mnt/d/` itd.) i pokrenite Loongcode.

   ```bash
   cd /mnt/c/Users/YourName/project
   loongcode
   ```

</Steps>

## Desktop aplikacija + WSL Server

Ako preferirate Loongcode Desktop aplikaciju, ali zelite da server radi u WSL-u:

1. **Pokrenite server u WSL-u** sa `--hostname 0.0.0.0` da dozvolite vanjske konekcije:

   ```bash
   loongcode serve --hostname 0.0.0.0 --port 4096
   ```

2. **Povezite Desktop aplikaciju** na `http://localhost:4096`

:::note
Ako `localhost` ne radi u vasem setupu, povezte se preko WSL IP adrese (u WSL-u: `hostname -I`) i koristite `http://<wsl-ip>:4096`.
:::

:::caution
Kada koristite `--hostname 0.0.0.0`, postavite `LOONGCODE_SERVER_PASSWORD` da zastitite server.

```bash
LOONGCODE_SERVER_PASSWORD=your-password loongcode serve --hostname 0.0.0.0
```

:::

## Web klijent + WSL

Za najbolje web iskustvo na Windowsu:

1. **Pokrenite `loongcode web` u WSL terminalu** umjesto u PowerShell-u:

   ```bash
   loongcode web --hostname 0.0.0.0
   ```

2. **Otvorite iz Windows browsera** na `http://localhost:<port>` (Loongcode ispisuje URL)

Pokretanje `loongcode web` iz WSL-a osigurava ispravan pristup fajl sistemu i terminalsku integraciju, a i dalje je dostupno iz Windows browsera.

## Pristup Windows fajlovima

WSL moze pristupiti svim Windows fajlovima kroz `/mnt/` direktorij:

- Disk `C:` ŌåÆ `/mnt/c/`
- Disk `D:` ŌåÆ `/mnt/d/`
- I tako dalje...

Primjer:

```bash
cd /mnt/c/Users/YourName/Documents/project
loongcode
```

:::tip
Za najgladje iskustvo, razmislite da klonirate/kopirate repo u WSL fajl sistem (npr. pod `~/code/`) i tu pokrenete Loongcode.
:::

## Savjeti

- Drzite Loongcode u WSL-u za projekte na Windows diskovima - pristup fajlovima je jednostavan
- Koristite VS Code [WSL ekstenziju](https://code.visualstudio.com/docs/remote/wsl) uz Loongcode za integrisan tok rada
- Vase Loongcode konfiguracije i sesije cuvaju se u WSL okruzenju na `~/.local/share/loongcode/`
