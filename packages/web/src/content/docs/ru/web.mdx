loongcode может работать как веб-приложение в вашем браузере, обеспечивая такой же мощный опыт кодирования AI без необходимости использования терминала.

![loongcode Web — новый сеанс](../../../assets/web/web-homepage-new-session.png)

## Начало работы

Запустите веб-интерфейс, выполнив:

```bash
loongcode web
```

Это запустит локальный сервер `127.0.0.1` со случайным доступным портом и автоматически откроет loongcode в браузере по умолчанию.

:::caution
Если `LOONGCODE_SERVER_PASSWORD` не установлен, сервер будет незащищен. Это подходит для локального использования, но его следует настроить для доступа к сети.
:::

:::tip[Пользователи Windows]
Для получения наилучших результатов запустите `loongcode web` из [WSL](/docs/windows-wsl), а не из PowerShell. Это обеспечивает правильный доступ к файловой системе и интеграцию терминала.
:::

## Конфигурация

Вы можете настроить веб-сервер с помощью CLI-флагов или в файле [config file](/docs/config).

### Порт

По умолчанию loongcode выбирает доступный порт. Вы можете указать порт:

```bash
loongcode web --port 4096
```

### Имя хоста

По умолчанию сервер привязывается к `127.0.0.1` (только локальный хост). Чтобы сделать loongcode доступным в вашей сети:

```bash
loongcode web --hostname 0.0.0.0
```

При использовании `0.0.0.0` loongcode будет отображать как локальные, так и сетевые адреса:

```
  Local access:       http://localhost:4096
  Network access:     http://192.168.1.100:4096
```

### Обнаружение mDNS

Включите mDNS, чтобы ваш сервер был доступен для обнаружения в локальной сети:

```bash
loongcode web --mdns
```

Это автоматически устанавливает имя хоста `0.0.0.0` и объявляет сервер как `loongcode.local`.

Вы можете настроить доменное имя mDNS для запуска нескольких экземпляров в одной сети:

```bash
loongcode web --mdns --mdns-domain myproject.local
```

### CORS

Чтобы разрешить дополнительные домены для CORS (полезно для пользовательских интерфейсов):

```bash
loongcode web --cors https://example.com
```

### Аутентификация

Чтобы защитить доступ, установите пароль, используя переменную среды `LOONGCODE_SERVER_PASSWORD`:

```bash
LOONGCODE_SERVER_PASSWORD=secret loongcode web
```

Имя пользователя по умолчанию — `loongcode`, но его можно изменить с помощью `LOONGCODE_SERVER_USERNAME`.

## Использование веб-интерфейса

После запуска веб-интерфейс предоставляет доступ к вашим сеансам loongcode.

### Сессии

Просматривайте свои сеансы и управляйте ими с главной страницы. Вы можете видеть активные сеансы и начинать новые.

![loongcode Web — активный сеанс](../../../assets/web/web-homepage-active-session.png)

### Статус сервера

Нажмите «Просмотреть серверы», чтобы просмотреть подключенные серверы и их статус.

![loongcode Web — см. Серверы](../../../assets/web/web-homepage-see-servers.png)

## Подключение терминала

Вы можете подключить TUI терминала к работающему веб-серверу:

```bash
# Start the web server
loongcode web --port 4096

# In another terminal, attach the TUI
loongcode attach http://localhost:4096
```

Это позволяет вам одновременно использовать веб-интерфейс и терминал, используя одни и те же сеансы и состояние.

---

## Конфигурационный файл

Вы также можете настроить параметры сервера в файле конфигурации `loongcode.json`:

```json
{
  "server": {
    "port": 4096,
    "hostname": "0.0.0.0",
    "mdns": true,
    "cors": ["https://example.com"]
  }
}
```

CLI-флаги имеют приоритет над настройками файла конфигурации.
