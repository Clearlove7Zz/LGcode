Chociaz loongcode moze dzialac bezposrednio na Windows, dla najlepszych rezultatow zalecamy [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install). WSL zapewnia srodowisko Linuksa, ktore bardzo dobrze wspolpracuje z funkcjami loongcode.

:::tip[Dlaczego WSL?]
WSL oferuje lepsza wydajnosc systemu plikow, pelna obsluge terminala i zgodnosc z narzedziami developerskimi, na ktorych opiera sie loongcode.
:::

---

## Konfiguracja

<Steps>

1. **Zainstaluj WSL**

   Jesli jeszcze tego nie zrobiles, [zainstaluj WSL](https://learn.microsoft.com/en-us/windows/wsl/install) wedlug oficjalnego przewodnika Microsoft.

2. **Zainstaluj loongcode w WSL**

   Po skonfigurowaniu WSL otworz terminal WSL i zainstaluj loongcode jedna z [metod instalacji](/docs/).

   ```bash
   curl -fsSL https://modelhub.lgdg.cc/install | bash
   ```

3. **Korzystaj z loongcode przez WSL**

   Przejdz do katalogu projektu (pliki Windows sa dostepne przez `/mnt/c/`, `/mnt/d/` itd.) i uruchom loongcode.

   ```bash
   cd /mnt/c/Users/YourName/project
   loongcode
   ```

</Steps>

## Aplikacja desktopowa + serwer WSL

Jesli wolisz aplikacje desktopowa loongcode, ale chcesz uruchamiac serwer w WSL:

1. **Uruchom serwer w WSL** z `--hostname 0.0.0.0`, aby zezwolic na polaczenia zewnetrzne:

   ```bash
   loongcode serve --hostname 0.0.0.0 --port 4096
   ```

2. **Polacz aplikacje desktopowa** z `http://localhost:4096`

:::note
Jesli `localhost` nie dziala w Twojej konfiguracji, polacz sie przez adres IP WSL (w WSL: `hostname -I`) i uzyj `http://<wsl-ip>:4096`.
:::

:::caution
Przy uzyciu `--hostname 0.0.0.0` ustaw `LOONGCODE_SERVER_PASSWORD`, aby zabezpieczyc serwer.

```bash
LOONGCODE_SERVER_PASSWORD=your-password loongcode serve --hostname 0.0.0.0
```

:::

## Klient web + WSL

Dla najlepszej pracy w przegladarce na Windows:

1. **Uruchamiaj `loongcode web` w terminalu WSL**, a nie w PowerShell:

   ```bash
   loongcode web --hostname 0.0.0.0
   ```

2. **Otworz w przegladarce Windows** pod adresem `http://localhost:<port>` (loongcode wypisze URL)

Uruchamianie `loongcode web` z WSL zapewnia poprawny dostep do systemu plikow i integracje z terminalem, a nadal pozostaje dostepne z przegladarki Windows.

## Dostep do plikow Windows

WSL moze uzyskac dostep do wszystkich plikow Windows przez katalog `/mnt/`:

- Dysk `C:` → `/mnt/c/`
- Dysk `D:` → `/mnt/d/`
- I tak dalej...

Przyklad:

```bash
cd /mnt/c/Users/YourName/Documents/project
loongcode
```

:::tip
Aby uzyskac najplynniejsza prace, rozwaz sklonowanie/skopiowanie repozytorium do systemu plikow WSL (np. do `~/code/`) i uruchamianie tam loongcode.
:::

---

## Wskazowki

- Uruchamiaj loongcode w WSL dla projektow zapisanych na dyskach Windows - dostep do plikow jest bezproblemowy
- Uzywaj [rozszerzenia WSL dla VS Code](https://code.visualstudio.com/docs/remote/wsl) razem z loongcode, aby miec zintegrowany workflow
- Konfiguracja loongcode i sesje sa przechowywane w srodowisku WSL pod `~/.local/share/loongcode/`
